# 错误码规范

错误码由领域前缀、HTTP 语义和顺序号组成，例如 `AUTH-401-001`。

| 前缀 | 领域 | 典型错误 |
|---|---|---|
| `AUTH` | 认证授权 | Token 无效、权限不足、服务 JWT 无效 |
| `TENANT` | 租户隔离 | 租户缺失、跨租户访问 |
| `CONV` | 会话 | 会话不存在、重复消息、流已取消 |
| `KNOWLEDGE` | 知识 | 文档未发布、检索无依据、摄入失败 |
| `ORCH` | AI 编排 | 工作流状态冲突、工具参数无效、模型降级 |
| `MALL` | 电商中台 | 资源不存在、资源归属不符、业务规则拒绝 |
| `AFTERSALE` | 售后退款 | 确认过期、审批状态冲突、退款补偿失败 |
| `SYSTEM` | 基础设施 | 依赖不可用、超时、限流 |

错误消息面向调用方且不泄露实现细节；完整异常只写日志并关联 `requestId`。
