# AliAgent 项目现状

> 最后更新: 2026-06-18 | 分支: master

---

## 1. 项目定位

**AliAgent** 是一个基于阿里通义千问大模型的 AI 智能对话 Web 应用，支持多轮对话、会话管理、用户认证，并已集成 **RAG（检索增强生成）** — 文档上传后自动向量化，对话中自动检索知识库增强回答。

## 4. 数据库设计

### 4.1 表结构

**users** — 用户表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) PK | UUID |
| username | VARCHAR(50) UNIQUE | 用户名 |
| password | VARCHAR(255) | 密码（明文） |
| email | VARCHAR(100) UNIQUE | 邮箱（可选） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |
| deleted | SMALLINT | 软删除标记 |

**conversation** — 会话表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) PK | UUID |
| title | VARCHAR(255) | 默认"新对话" |
| deleted | SMALLINT | 软删除 |
| pinned | SMALLINT | 置顶标志 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

**message** — 消息表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) PK | UUID |
| conversation_id | VARCHAR(36) FK | 关联会话，ON DELETE CASCADE |
| role | VARCHAR(20) | user / assistant |
| content | TEXT | 消息内容 |
| metadata | JSONB | RAG 来源等扩展信息，默认 `'{}'` |
| created_at | TIMESTAMP | 创建时间 |

**document** — 文档元信息表 (RAG)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) PK | UUID |
| name | VARCHAR(255) | 文件名 |
| type | VARCHAR(50) | 文件类型 (pdf/txt/docx) |
| size | BIGINT | 文件大小 (bytes) |
| metadata | JSONB | 扩展元数据 |
| created_at | TIMESTAMP | 创建时间 |

**document_chunk** — 文档分块 + 向量表 (RAG)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) PK | UUID |
| document_id | VARCHAR(36) FK | 关联文档，ON DELETE CASCADE |
| content | TEXT | 分块文本内容 |
| embedding | vector(1024) | DashScope text-embedding-v3 向量 |
| section_title | VARCHAR(255) | 章节标题 |
| page_number | INT | 页码 |
| chunk_index | INT | 分块序号 |
| metadata | JSONB | 扩展元数据 |
| created_at | TIMESTAMP | 创建时间 |

索引: `idx_chunk_embedding` (HNSW, vector_cosine_ops)

### 4.2 关键配置

- **主键策略**: MyBatis-Plus `ASSIGN_UUID` (应用层生成 UUID)
- **逻辑删除**: `deleted=1` 表示已删除
- **自动填充**: `created_at` / `updated_at` 由 `MyMetaObjectHandler` 处理
- **向量索引**: pgvector HNSW 索引 + `vector_cosine_ops` 余弦相似度检索

## 5. 核心业务流程

### 5.1 流式 AI 对话 + RAG

```
用户输入消息
    ↓
前端: store.sendMessage()
    ↓
前端: GET /api/chat/stream?message=...&conversationId=... (SSE)
    ↓
后端: ChatController.streamChat()
    ↓
Agent.chat():
  1. ConversationService.getOrCreate() → 确保会话存在
  2. MessageService.getHistoryMessages() → 加载历史上下文
  3. MessageService.saveMessage() → 保存用户消息
  4. RAGPipeline.execute(message, conversationId):
     ├─ RuleStrategy → 判断是否触发检索
     ├─ NoOpQueryRewriter → 透传查询
     ├─ VectorRetriever → pgvector 向量检索 Top-K
     ├─ NoOpReranker → 直通
     ├─ SimpleContextBuilder → 拼接去重截断
     └─ BasicPromptBuilder → 构建 RAG Prompt
  5. 如果触发 RAG → 用 RAG Prompt 替换 user message
  6. chatClient.prompt().stream() → 流式调用千问
  7. doOnNext → 逐 chunk 推送 SSE
  8. doFinally → 保存 AI 回复 + sources 到 message.metadata
    ↓
前端: ReadableStream 消费 SSE → 逐字渲染 + RAG 来源卡片
```

### 5.2 用户认证流程

```
注册: POST /api/auth/register
  → AuthService 查重 → 插入 users 表 → 返回 userId

登录: POST /api/auth/login
  → AuthService 比对密码 → TokenService 生成 UUID Token
  → Redis 双向索引: token:{userId} 和 token_index:{token}
  → TTL 24h → 返回 Token

鉴权: TokenInterceptor (拦截 /api/**)
  → 从 Authorization: Bearer {token} 提取 Token
  → Redis 查询 token_index:{token} → 解析 userId
  → 写入 UserContext (ThreadLocal)

退出: DELETE /api/auth/logout
  → 清除 Redis 中的 Token 记录
```

### 5.3 RAG 文档摄入

```
文件上传: POST /api/rag/documents/upload
  → DocumentIngestService.ingest(MultipartFile):
    1. TikaDocumentReader 解析文本
    2. TokenTextSplitter 分块 (800 tokens, overlap 100)
    3. DashScopeEmbeddingModel 向量化 (1024维)
    4. INSERT INTO document (元信息)
    5. INSERT INTO document_chunk (分块 + embedding)
```

### 6.2 状态管理

不使用 Pinia/Vuex，采用纯 `reactive()` 对象 (`stores/index.ts`)：

| 字段 | 类型 | 说明 |
|------|------|------|
| authenticated | boolean | 是否已登录 |
| conversations | Conversation[] | 会话列表 |
| currentConversationId | string\|null | 当前会话 ID |
| messages | Message[] | 当前会话消息 (含 sources 解析) |
| isLoading | boolean | 加载状态 |
| streamingContent | string | 流式内容缓存 |
| sidebarOpen | boolean | 侧边栏展开状态 |
| theme | 'light'\|'dark' | 主题 |
| serviceOnline | boolean | 后端服务是否在线 |
| knowledgeBaseOpen | boolean | 知识库面板开关 |
| ragDocuments | RAGDocument[] | 知识库文档列表 |

### 6.3 设计系统

- **名称**: Apple HIG (Human Interface Guidelines)
- **主色调**: 系统蓝 (`#0a84ff` 暗色 / `#007aff` 浅色)
- **风格**: 简洁专业，信息优先，无多余装饰
- **双主题**: CSS 变量驱动 `html[data-theme]`，登录页品牌区永远深色
- **字体**: 系统字体栈 (`SF Pro / PingFang SC`)，不加载 Web 字体
