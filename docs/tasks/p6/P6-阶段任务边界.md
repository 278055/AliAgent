## 事务、安全与降级

创建申请、命令去重、Saga 状态和 Outbox 必须同一 MySQL 本地事务。退款、权益回滚和通知为独立步骤；后续失败不掩盖整体失败。可重试步骤写 `RETRY_PENDING`；退款已成功而权益失败写 `MANUAL_RECONCILIATION` 并产生日志、审计和待处理事件。日志脱敏，严禁记录 JWT、支付令牌、完整地址或未经脱敏订单。

订单租户必须先由集成冻结的 `OrderTenantResolver` 从 `order_tenant_binding` 解析；可信请求租户与解析结果不等、订单未绑定或绑定异常时必须拒绝。历史订单仅能由批准的运营批次回填，禁止以 `mall-default`、会员 ID 或当前请求租户推断；新订单事务写入绑定属于集成后续公共任务，未完成前不得接受其 P6 售后写命令。

测试资源只能用 `test-`/`rag-test-` 前缀；端到端产生的 `conversation`、`message`、`document`、`document_chunk` 与测试用户必须清理。mall 为 MySQL，不适用 PostgreSQL 空库迁移校验；若触及 PostgreSQL 服务，先按 `aliagent-database` Skill 做只读确认，DDL 只能用 Java JDBC。
