# P5 阶段任务边界

## 基线和公共边界

- 集成分支 `codex/integration-p5` 基于 P4 `bf6a3c4`。P5 仅只读编排：普通、RAG、订单、物流、转人工；任何退款、取消、售后、审批或业务写操作均拒绝。
- 集成会话独占：根 POM、编排服务 POM、`application*.yml`、`db/migration/`、`contracts/`、`orchestration/contract/`。实施者发现缺口只能提出集成变更请求。
- 实际 v1 队列为 `ai.reply.requested.v1`，conversation-service 直接发送到 queue，未配置 exchange/routing key；v1 负载仅 conversationId/messageId/requestId。v2 另用 `ai.reply.requested.v2`，增加 generationId，v1 保持兼容。

## 实际依赖

- 回写：`POST /internal/api/v1/conversations/{conversationId}/generations/{generationId}/chunks`，body 为 messageId/chunkIndex/delta/finalChunk/finishReason；重复或倒退 chunkIndex 被忽略。v1 无 generationId，必须记录等待/不兼容，禁止猜测。
- 知识：`POST /api/v1/knowledge/retrieval:query`，body `{query,topK}`，返回 documentId/versionId/chunkId/content/score/citation；要求可信身份头、授权快照及服务 JWT。
- mall：`GET /api/v1/internal/mall/orders/{orderId}` 与 `.../{orderId}/logistics`；要求服务 JWT 和可信身份头；仅合规 MEMBER/STAFF 可读。
- `ExecutionContext` 已冻结 tenantId、subjectId、subjectType、roles、permissions、traceId、requestId、conversationId、messageId、generationId、authorizationSnapshotId；公共 ports 已冻结。

## 所有权

| 任务 | 允许目录 | 职责 |
|---|---|---|
| P5-A | `orchestration/core/`、`routing/`、`workflow/`、`messaging/` 和测试 | MQ、Inbox、状态机、路由、五工作流、恢复 |
| P5-B | `orchestration/adapter/`、`tool/` 和测试 | Mock/DashScope、knowledge/mall/conversation 适配 |
| P5-C | `orchestration/governance/`、`quota/`、`resilience/`、`audit/` 和测试 | 版本、灰度、配额、Sentinel、审计、降级 |

三者禁止修改公共文件、其他任务目录、conversation-service、knowledge-service、mall、旧单体、deploy、frontend。

## 安全和验收

- Inbox 用 eventId+consumer；状态、工具、模型审计保证重启不重放已完成调用。固定 prompt/workflow/model/rule/knowledge 实际版本。
- 模型不得猜测订单、物流、价格、库存、退款、审批；知识或 mall 故障不虚构事实；模型失败安全普通提示或转人工。WRITE 工具在 P5 拒绝。默认 Mock，只有 provider=dashscope 且 `DASHSCOPE_API_KEY` 存在才真调，禁止提交密钥。
- 测试数据前缀 `test-p5-a-*`、`test-p5-b-*`、`test-p5-c-*`，先删子表再删父表。交付提交 SHA、文件、验证、集成请求/风险。

验收：执行指定 Maven 命令、`git diff --check` 和 `git status --short`。
