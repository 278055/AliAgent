# P1 冻结清单

P0 验收通过后，P1 只能在下列边界内实施。

| 项目 | 冻结值 |
|---|---|
| 服务名 | `gateway-service`、`conversation-service`、`ai-orchestration-service`、`knowledge-service`、`evaluation-service`、`insight-service` |
| PostgreSQL 数据库 | `conversation_db`、`orchestration_db`、`knowledge_db`、`evaluation_db`、`insight_db` |
| 数据库归属 | 每个服务一个独立 PostgreSQL 账号；禁止跨库查询；`mall_db` 归 mall/MySQL。 |
| 契约版本 | HTTP 从 `/api/v1` 起；事件信封 `eventVersion: 1`。 |
| 端口策略 | 本地端口由 P1-C Compose 统一分配并写入 `deploy/README.md`；服务代码不得硬编码端口。 |
| 目录所有者 | P1-A：根 `pom.xml`、`contracts/`、CI；P1-B：`services/`；P1-C：`deploy/`、`frontend/`。 |
| 禁止项 | P1 不迁移聊天/RAG 业务，不引入售后写操作，不删除旧接口。 |

P1 集成验收必须验证：Compose 基础设施可启动、服务健康接口可路由、服务只能连接所属数据库、未授权服务 JWT 被拒绝、契约校验可运行。
